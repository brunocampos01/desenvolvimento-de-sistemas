 
Trabalho 2 - Aplicação Web
A tarefa consiste em desenvolver um sistema Web para marcação de consultas no Hospital Universitário (HU). O sistema deve ser implementado usando EJB, JSF e JPA. As entidades devem ser mantidas em um banco de dados.

O Sistema de Marcação de Consultas
O sistema deverá ser acessado utilizando um navegador Web, devendo exigir autenticação do usuário. Os funcionários do hospital, devidamente autenticados, poderão informar os horários disponíveis para consulta no seu setor. Os funcionários também serão responsáveis por fazer o cadastramento inicial dos pacientes. Os pacientes já cadastrados poderão acessar o sistema, visualizar os horários disponíveis para consulta e agendar consultas com profissionais de diversas especialidades. Deve haver um limite de uma consulta marcada por especialidade por paciente. As consultas marcadas anteriormente pelo paciente serão exibidas na sua página inicial, após a realização do login. O paciente poderá, na página inicial, cancelar consultas já marcadas com pelo menos 24 horas de antecedência.

Inicialmente devem ser criadas as tabelas no banco de dados. Crie tabelas para todas as entidades do sistema. Em seguida crie os beans de entidade a partir das tabelas do banco de dados. Na etapa seguinte, crie um ou mais beans de sessão com os métodos de negócio. Por fim, crie a camada web da aplicação, que deve conter facelets que permitirão a execução de cada funcionalidade prevista, e beans gerenciados que utilizarão os métodos de negócio disponibilizados pelos beans de sessão. 

Na criação dos facelets, sugerimos que seja usada a biblioteca de componentes PrimeFaces. Para isso, adicione a biblioteca ao projeto web e adicione as tags correspondentes aos componentes, descritos na documentação disponível aqui. 

Avaliação
A atividade deve ser desenvolvida em duplas. A apresentação deve ser realizada até o dia 19/10. Os dois integrantes da equipe devem estar presentes no momento da apresentação. Será verificado o funcionamento do programa e em seguida os alunos devem responder oralmente a questões formuladas pelo professor sobre o código fonte do programa. Podem ser atribuídas notas diferentes aos integrantes de uma equipe, dependendo das respostas às perguntas sobre o código do programa efetuadas pelo professor. Caso um dos integrantes não esteja presente ou demonstre não conhecer o código do programa, este ficará com nota zero na atividade. Em caso de cópia do código de outra equipe, ambas terão nota igual a zero.
